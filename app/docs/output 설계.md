# 안전신문고 각 항목에 대한 분석 및 각 항목 제공 계획

0. **위반 이벤트 유형:**
    - ex. 안전거리 미확보
1. **자동차·교통 위반 신고 유형:** 
    - ex. 교통위반(교통단속 포함)
2. **사진/동영상(첨부):** 
    - 동영상 1개, 그 중 핵심적인 사진 1장, 차량을 crop한 사진 1장, 그 중 번호판을 crop한 사진 1장 총 4장
3. **신고 발생 지역: ← 이게 신고 단계에서 사실상 가장 핵심적인 개발 요소라고 생각함.**
    - 우리 프로젝트에선 gps가 달린 카메라라, 사진 정보에 gps가 저장된다고 가정.
    - 일단 이 부분은 나중에 구현, 일단은 gps(위치 정보)는 빼고 구현
            
4. **제목(최소 2자, 최대 150자 작성 가능)**
    - llm이 위에 “1. 자동차·교통 위반 신고 유형**”**과, 우리가 판단한 **“**위반 항목” 정보를 받아서, 짜준 결과를 사용.
5. **신고 내용(최대 1600자 작성 가능)**
    - 제목과 마찬가지로, llm이 위에 “1. 자동차·교통 위반 신고 유형**”**과, 우리가 판단한 **“**위반 항목” 정보를 받아서, 짜준 결과를 사용.
    - 제목과 신고 내용은 llm 프롬프팅을 적당히 해야 할 듯함.
        - 사용자에게 “제목 및 신고 내용 항목을 llm으로 작성하시겠습니까?”라고 물어보고, 사용자가 사용하겠다고 답변하면 llm api key를 입력받아 주어진 프롬프트를 기반으로 llm이 답변
6. **차량 번호:**
    - 우리의 번호판 crop 및 OCR model이 뽑아낸 차량 번호를 그대로 사용
7. **발생 일자:**
    - 아래 발생 시각에서 일자만 추출.
8. **발생 시각:**
    - 동영상에 저장된 시간 정보를 추출한 다음,
    - 그 정보를 가지고 단위시간 당 프레임 수를 계산해서 그 값을 더하면,
    - 우리가 탐지한 구간의 첫 번째 프레임의 촬영 시각을 알 수 있음. 해당 시각을 작성.
9. **휴대전화:**
    - 사용자가 미리 입력한 내용 그대로
10. **인증번호:**
    - 이건 사용자가 입력해야 하지 않을까?
11. **신고 내용 공유:**
    - 일단 아니요를 기본으로 제공
12. **인적 사항:**
    1. 사용자가 미리 입력한 내용 그대로
